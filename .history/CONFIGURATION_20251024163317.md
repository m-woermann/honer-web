# Configuration Guide

This website is **100% configuration-driven** through the `src/config/speakers.json` file. You can change models, add products, modify specifications, and update all content by editing this single JSON file.

## How It Works

All pages dynamically load data from `src/config/speakers.json`:
- **Main page (/)**: Displays speaker stacks from the `stacks` array
- **Products page (/products)**: Lists all speaker types from `speakerTypes`
- **Speaker detail page (/speaker-detail)**: Shows individual speaker specs

## File Structure

```
src/config/speakers.json
```

```json
{
  "speakerTypes": {
    "top": {
      "modelPath": "/speaker.glb",
      "name": "Your Speaker Name",
      "description": "Your speaker description",
      "specs": {
        "Power": "500W RMS",
        "Frequency Response": "90Hz - 20kHz",
        ...
      }
    },
    "sub": {
      "modelPath": "/sub.glb",
      ...
    }
  },
  "stacks": [
    {
      "name": "Small System",
      "description": "System description",
      "spacing": 3000,
      "speakers": [
        {
          "type": "sub",
          "x": -185,
          "y": 0,
          "z": 0,
          "rotationY": 1.5707963267948966,
          "onGround": true
        }
      ]
    }
  ]
}
```

## How to Add/Change Products

### 1. Add a New Speaker Type

Edit `src/config/speakers.json` and add a new entry to `speakerTypes`:

```json
"speakerTypes": {
  "top": { ... },
  "sub": { ... },
  "newSpeaker": {
    "modelPath": "/new-speaker.glb",
    "name": "New Speaker Name",
    "description": "Description of the new speaker",
    "specs": {
      "Power": "1000W RMS",
      "Frequency Response": "30Hz - 18kHz",
      "Drivers": "12\" Woofer + 2\" Horn",
      "Connections": "XLR",
      "Custom Field": "Any value you want"
    }
  }
}
```

**Important:** Place your GLB file in the `public/` folder (e.g., `public/new-speaker.glb`)

### 2. Modify Existing Specs

Simply edit the values in the `specs` object:

```json
"specs": {
  "Power": "2000W RMS",  // Changed from 500W
  "New Spec": "New Value" // Added new specification
}
```

You can add, remove, or modify any spec fields - they will automatically appear on the products and detail pages.

### 3. Change 3D Models

Replace the GLB file path in `modelPath`:

```json
"modelPath": "/my-new-model.glb"
```

Then place `my-new-model.glb` in the `public/` folder.

### 4. Add a New Stack Configuration

Add a new object to the `stacks` array:

```json
{
  "name": "Extra Large System",
  "description": "12 Subwoofers with quad tops",
  "spacing": 3500,
  "speakers": [
    {
      "type": "sub",
      "x": 0,
      "y": 0,
      "z": 0,
      "rotationY": 0,
      "onGround": true
    }
  ]
}
```

## No Hard-Coded Values

✅ **All content is from speakers.json:**
- Product names
- Descriptions
- Specifications
- 3D model paths
- Stack configurations
- Speaker positions

❌ **Nothing is hard-coded in the pages:**
- No fixed product lists
- No static model paths
- No manually entered specs
- No fixed speaker arrangements

## Testing Changes

1. Edit `src/config/speakers.json`
2. Add GLB files to `public/` folder if needed
3. Save the file
4. Refresh your browser
5. All changes will be reflected immediately

## Troubleshooting

### Models not appearing?

1. Check browser console (F12) for errors
2. Verify GLB files exist in `public/` folder
3. Ensure `modelPath` matches the filename exactly
4. Check that paths start with `/` (e.g., `/speaker.glb`)

### Specs not showing?

1. Verify the `specs` object syntax is valid JSON
2. Check for missing commas or quotes
3. Ensure all spec values are strings

### Console Commands

Open browser console and type:
```javascript
// Check if models are loading
console.log('Model containers:', document.querySelectorAll('.product-image').length);
```
